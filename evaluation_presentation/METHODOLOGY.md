# Method Comparison — Deep Creativity on 2-and-6 Generation

This folder benchmarks several image-generation strategies that aim to produce
MNIST-like digits that look simultaneously like a **2** and a **6**. Each method
contributes 10,000 samples; we score them with the **Deep Creativity (DC)** metric
and report mean/std plus visualizations of the top-25 and a random-25 sample
from each method.

## Deep Creativity metric

`src/metrics.py :: DeepCreativity` returns

```
DC(x) = exp( w_V · log V(x) + w_N · log N(x) + w_S · log S(x) )
```

with all weights set to 1. The three components:

| Component | Definition | Scoring model |
|---|---|---|
| `V(x)` — value | "Does this look like a real digit?" | GAN discriminator (`use_discriminator=True`) — `src/GAN/checkpoints/discriminators/discriminator_last.pth` |
| `N(x)` — novelty | Binary entropy of `P(2|x)` vs `P(6|x)` (renormalized over the two), times relevance `P(2|x) + P(6|x)`. Maximized at the 2/6 decision boundary. | EMNIST 10-class classifier — `src/mnist_models/digit_classifier/emnist_mixup_classifier.pth` |
| `S(x)` — surprise | `1 − exp(−λ·KL(q(z|x) ‖ N(0, I)))` from the basic mnist VAE encoder. Penalizes near-mode-collapse images. | VAE encoder — `full_VAE.pth` |

We use the **discriminator-based value** (the alternative is a separate
EMNIST-trained binary "is_digit" CNN; both ship with the scorer). Discriminator
mode is the harsher of the two, so reported DC values are conservative.

All four scoring models are frozen (`.eval()`, `requires_grad=False`).

## Methods compared

### 1. `vae_interp` — latent interpolation in the basic VAE
Generated by `generate_vae_interp.py`. For each sample:

1. Pick a random MNIST 2 (`x₂`) and a random MNIST 6 (`x₆`).
2. Encode each to latent **means** (no reparameterization noise):
   `μ₂ = VAE.Encoder(x₂).mean`, `μ₆ = VAE.Encoder(x₆).mean`.
3. Sample `a ~ Uniform(−0.1, +0.1)` and combine
   `z = ((1 + a)·μ₂ + (1 − a)·μ₆) / 2`.
4. Decode to a 28×28 image, clamp to `[0, 1]`.

The narrow band on `a` keeps samples close to the geometric midpoint of the two
class manifolds while still adding pair-to-pair variation. Source dataset:
`torchvision.datasets.MNIST` train split (5,958 twos and 5,918 sixes available).
Seed: `torch.Generator().manual_seed(2026)`.

### 2. `diffusion_avg` and `diffusion_prod` — compositional classifier-free guidance
Generated by `src/diffusion/v3_score/generate_images.py` against a class-conditional
DDPM checkpoint (`src/diffusion/checkpoints/ddpm_class`, EMA weights). At every
denoising step we make one batched UNet call producing `[ε_{c=2}, ε_{c=6}, ε_∅]`.
With deltas `d_i = ε_{c_i} − ε_∅` and guidance `w = 1.5`:

* **`average`:** `ε = ε_∅ + w · 0.5 · (d_1 + d_2)` — geometric average of the two
  conditionals; matches a normal CFG step in magnitude.
* **`product`:** `ε = ε_∅ + w · (d_1 + d_2)` — composable diffusion (Liu et al.):
  samples from `p(x|2) · p(x|6)`, the *intersection* of the two class manifolds.

10,000 samples per mode, distributed across GPUs 1–3 via `torch.multiprocessing.spawn`,
batch size 200 per GPU (~5 GB peak), 1,000 timesteps, output cropped from 32×32
to the central 28×28.

### 3. Friend's two VAE-based methods (placeholder)
Not yet integrated. Drop the two `.pt` files (10000 × 1 × 28 × 28, values in
`[0, 1]`) into `evaluation_presentation/` and uncomment the corresponding entries
in `METHODS` inside `compare_methods.ipynb`.

## Scoring procedure

`compare_methods.ipynb` executes these steps:

1. Load all four scoring models on `cuda:0` and instantiate
   `DeepCreativity(use_discriminator=True)`.
2. For every method whose `.pt` exists on disk:
   * Load the tensor, ensure shape `(N, 1, 28, 28)`, clamp to `[0, 1]`.
   * Score in batches of 256 with `@torch.no_grad()`.
3. Per method, record DC `min`, `max`, `mean`, `std`, `median`.
4. Pick the top-25 indices by descending DC; pick a random-25 with
   `numpy.random.default_rng(2026)` (fixed seed → reproducible across reruns).

## Outputs

* `vae_interp_samples.pt` — 10,000 latent-interpolated samples.
* `all_scores.pt` — dict of `{method_name: DC scores tensor}`.
* `dc_mean_std_bar.png` — bar chart of mean DC per method with ±1 std error bars.
* `top25_<method>.png`, `rand25_<method>.png` — 5×5 grids with per-image DC scores
  printed as titles.
* Console table with per-method N, mean, std, median, min, max.

## Reproducibility notes

* VAE-interp uses MNIST orientation (the VAE was trained on MNIST). Other
  methods produce MNIST-orientation images natively.
* Diffusion seeds: each rank uses
  `torch.manual_seed(1234 + rank·7919 + hash(mode) % 10_000)`, so different ranks
  and modes do not produce duplicates.
* `clamp(0, 1)` is applied before scoring everywhere — diffusion outputs are not
  bounded by construction.
* All scoring is performed against the same four frozen models, so DC is
  directly comparable across methods.
